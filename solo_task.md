## 기준

아래 조건을 만족하면 지금 혼자 선작업 가능하다.

- 더미 데이터로 화면을 만들 수 있다
- API 없이도 UI와 상태를 먼저 만들 수 있다
- 나중에 데이터만 꽂으면 된다
- 팀원 코드와 직접 충돌 가능성이 낮다

## 1순위

### 1. 프로필 페이지 틀

추천 이유:
- 혼자 만들기 가장 쉽다
- 나중에 API 붙이기 쉽다
- 평가 때 보여주기 좋다

미리 만들 것:
- `ProfilePage`
- `ProfileCard`
- `ProfileEditForm`
- 아바타 표시 영역
- 소개글 영역
- 활동 요약 섹션

미리 정할 것:
- 유저 데이터 타입 초안
- 읽기 모드 / 수정 모드 전환 방식
- 아바타가 없을 때 기본 UI

더미 데이터 예시:
- 닉네임
- 이메일
- 소개글
- 리뷰 수
- 찜 수
- 팔로우 수

완료 기준:
- 프로필 조회 화면이 더미 데이터로 렌더링된다
- 수정 폼 UI가 분리되어 있다
- 나중에 API 응답만 넣으면 붙일 수 있다

### 2. 마이페이지 틀

추천 이유:
- 섹션이 많아서 미리 틀 잡아두면 뒤가 편하다
- 실제 데이터가 없어도 구조 작업이 가능하다

미리 만들 것:
- `MyPageDashboard`
- 내 리뷰 섹션
- 찜 목록 섹션
- 최근 활동 섹션
- 빈 상태 UI

미리 정할 것:
- 카드형 레이아웃인지 리스트형인지
- 탭 구조를 쓸지 섹션 구조를 쓸지
- 모바일에서 어떻게 줄일지

완료 기준:
- 더미 데이터 기준으로 마이페이지 레이아웃이 완성된다
- 데이터가 없을 때 빈 상태 UI가 보인다

## 2순위

### 3. 취향 선택 페이지

추천 이유:
- Week 2 작업이지만 UI는 지금 미리 만들 수 있다
- 백엔드 없이도 선택 상태 관리 구현 가능하다

미리 만들 것:
- `OnboardingPage`
- 장르 선택 카드
- 작품 선택 카드
- 선택 완료 버튼

미리 정할 것:
- 최소 선택 개수
- 선택 완료 버튼 활성화 조건
- 장르/작품 선택 방식

완료 기준:
- 장르 선택이 된다
- 작품 선택이 된다
- 선택 수에 따라 버튼 상태가 바뀐다

### 4. 리뷰 컴포넌트

추천 이유:
- 화면 단위보다 재사용 컴포넌트가 많아서 미리 해두면 좋다
- 더미 데이터로 거의 전부 가능하다

미리 만들 것:
- `StarRating`
- `ReviewCard`
- `ReviewForm`
- `ReviewList`

미리 정할 것:
- 별점 입력 방식
- 스포일러 표시 방식
- 정렬 버튼 UI
- 수정/삭제 버튼 위치

완료 기준:
- 더미 리뷰 목록이 렌더링된다
- 별점 입력/표시가 된다
- 리뷰 작성 폼이 있다

## 3순위

### 5. 비밀번호 변경 폼

미리 만들 것:
- 현재 비밀번호 입력
- 새 비밀번호 입력
- 새 비밀번호 확인 입력
- 검증 문구
- 성공/실패 메시지 자리

완료 기준:
- 프론트 유효성 검사가 된다
- API 없이도 제출 흐름 UI가 완성된다

### 6. 계정 설정 화면

미리 만들 것:
- 프로필 수정 섹션
- 비밀번호 변경 섹션
- 계정 탈퇴 버튼 자리
- 2FA 설정 자리

완료 기준:
- 설정 화면 레이아웃이 완성된다
- 나중에 기능만 붙이면 된다

### 7. 공용 UI 컴포넌트

미리 만들 것:
- 버튼 variants
- 입력 필드 래퍼
- 섹션 카드
- 빈 상태 컴포넌트
- 에러 메시지 박스
- 로딩 스켈레톤

완료 기준:
- 여러 페이지에서 재사용 가능한 기본 UI가 준비된다
